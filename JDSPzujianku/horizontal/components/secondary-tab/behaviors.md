# 二级 Tab · 行为规范

> design.md → [index](./design.md) · 同 bundle: [spec](./spec.md) · [variants](./variants.md) · schema: [ai-schema.yaml](./ai-schema.yaml)

## 应用场景

| 场景 | 使用形态 | 说明 |
|---|---|---|
| 内容流二级分类 | 左右滑动 | 在同一内容页内切换精选、时尚、科技、数码等分类 |
| 分类较多 | 左右滑动 + 加号 | 横向列表只展示部分分类，更多分类通过入口展开 |
| 分类总览 | 全部 tab | 展示完整分类列表，可能为加号展开后的状态 |
| 单状态说明 | 选中态 / 非选中态 | 用于规范单个筛选项样式 |

## 交互

| 交互对象 | 行为 | 反馈 |
|---|---|---|
| tab item | 点击切换当前分类 | 当前 item 变为选中态，原选中项恢复非选中态 |
| 横向滚动区 | 左右滑动浏览分类 | 列表横向滚动，右侧加号保持固定 |
| 加号入口 | 展开更多分类或进入分类管理 | 行为待产品确认；推荐展开全部 tab 面板 |
| 全部 tab item | 点击切换分类并收起面板 | 如果全部 tab 是展开态，点击后回到横滑模式 |

## 滚动规则

| 规则 | 说明 |
|---|---|
| 右侧入口固定 | 加号入口不参与横向滚动 |
| 当前选中可见 | 切换后应滚动到让选中项可见的位置 |
| 边界不回弹过度 | 滚动边界保持自然，不出现空白长距离 |
| 横向列表单行 | item 不换行，不纵向扩展 |
| 滚动区宽度 | 当前 Relay 为 331 px，需保留右侧入口空间 |

## 溢出与降级

| 场景 | 降级 |
|---|---|
| 分类数量少 | 可隐藏加号入口，使用全部 tab 行样式 |
| 分类数量多 | 横向滚动，保留加号入口 |
| 文案过长 | 优先自适应 item 宽度；超过最大宽度时截断 |
| 无加号能力 | 隐藏加号入口，滚动区可扩展至容器宽度 |
| icon-add 缺失 | 使用全局加号图标兜底，不新增临时图形 |

## 推荐动效

| 场景 | 参数 |
|---|---|
| 选中态切换 | 150-180ms background / color transition |
| 横向滚动到选中项 | 180-240ms ease-out |
| 加号展开全部 tab | 180ms opacity + translateY |
| 点击反馈 | item opacity 0.85 或轻微背景变化 |

## 禁止规则

- 不要让加号入口跟随列表横向滚动。
- 不要用多行文字承载分类名称。
- 不要在选中态使用强实底红色。
- 不要把全部 tab 展示和横滑列表同时堆叠在生产页面中。
- 不要在二级 Tab 中承载复杂筛选条件。
