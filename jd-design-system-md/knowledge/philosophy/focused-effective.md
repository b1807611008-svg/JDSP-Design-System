# 🎯 聚焦实效

> 京东是电商平台,不是艺术馆。每一个像素都要为转化、信息获取效率、用户决策路径让路。
>
> **当美感和转化冲突时,转化优先**。这不是设计师没追求,是商业产品的常识。

---

## 1. 核心主张

| 主张 | 具体表现 |
|---|---|
| 信息密度优先 | 一屏多卡(双列流)、价格突出、关键卖点前置 |
| 决策路径最短 | 加购 / 立购 一键直达,不绕弯路 |
| 视觉服务于内容 | 装饰元素让路给商品图、价格、CTA |
| 每个像素有理由 | 不放置"看起来好看但用户不需要"的元素 |

---

## 2. 反面是什么(不该做的)

| ❌ 反面行为 | 解释 |
|---|---|
| 装饰过度 | 大块 banner 抢占商品曝光,首屏看不到第二商品 |
| 视觉自嗨 | 设计师追求形式美感而忽略用户决策成本 |
| 路径过长 | 加购前必须填三个表单 / 看一段动画 |
| 信息密度不足 | 大留白看起来"高级"但牺牲屏效 |

---

## 3. 决策路径量化

**京东 APP 关键路径平均屏数**:
| 路径 | 当前平均屏数 | 目标 |
|---|---|---|
| 进入 APP → 商品 PDP | 2.3 屏 | ≤ 2 屏 |
| PDP → 加购 | 1.0 屏(同屏) | 1 屏 |
| 购物车 → 提交订单 | 2.1 屏 | ≤ 2 屏 |
| 提交订单 → 支付完成 | 2.5 屏 | ≤ 3 屏 |

**总转化路径**:打开 APP → 完成支付,平均 7.9 屏。每多 1 屏,流失约 8-12%(2025 全年漏斗数据)。

> **设计师的隐形 KPI**:你的设计在不在让这条路径变更短?

---

## 4. 信息密度的边界

聚焦实效 ≠ 堆砌信息。**信息密度的边界是"用户能扫到 + 不焦虑"**。

| 场景 | 推荐密度 | 反例 |
|---|---|---|
| 首页 / 频道页 | 高(双列流,卡 350-400pt) | 单列大图占满屏 |
| PDP 商品图区 | 中(主图 + 缩略图列) | 信息全堆主图上 |
| 表单 / 设置页 | 低(疏密有致,group 间留白) | 表单挤成一坨 |
| 错误 / 空状态 | 极低(图 + 一句话) | 错误页堆 5 条建议 |

---

## 5. 引用此原则的场景

- **PDP 评审**:首屏内容是否被装饰元素挤压?
- **购物车 / 结算评审**:是否有可去除的步骤?
- **首页 Banner 评审**:Banner 与下方商品流的视觉博弈是否合理?
- **大促设计评审**:节庆装饰是否影响商品识别?(品牌表现 vs 聚焦实效冲突时,看场景:大促首屏品牌优先,大促 PDP 仍以转化优先)

---

## 6. 与其他原则的张力

**与"品牌表现"的张力**:
- 大促首屏需要强品牌氛围 → 接受短暂的"装饰优先"
- 但 PDP / 结算 / 支付 → 永远聚焦实效优先

**协调机制**:Zone 5 horizontal/brand/promotion 中标注"品牌优先区"和"转化优先区",视觉资产分开做。

---

## 7. 历史踩坑

| 时间 | 事件 | 教训 |
|---|---|---|
| 2024-Q3 | 某事业部首页改为单列大图 | 屏效下降 22%,3 周回滚 |
| 2025-Q1 | PDP 加 3 步引导动画 | 加购率 -7%,2 周回滚 |
| 2025-Q4 | 双 11 视觉过度抢戏(全屏礼花) | 商品识别度下降,改为局部装饰 |

每次都是一次"美感-实效"博弈的实战教训。
