# 🤝 多元包容

> 京东 APP 用户从 18 岁到 80 岁,从一线城市到县乡,从健康人到视障听障老花眼。**默认人群假设是设计的最大盲区**。
>
> 包容不是慈善,是商业现实。中国 60 岁以上用户已超 3 亿,视障 / 弱视 / 色盲 / 轻度认知障碍人群超过 1 亿。

---

## 1. 核心主张

| 主张 | 具体表现 |
|---|---|
| 默认满足 WCAG 2.1 AA | 对比度 / 触达 / 屏幕阅读 / 键盘 / 减少动效 |
| 适老模式不是另一个 APP | 在标准模式下保留切换到大字 / 简化模式的能力 |
| 极端字号不破坏布局 | 系统字号 85%-150% 全程可用 |
| 无障碍是前置条件 | 不是"以后再加",是组件出生时就接入 |

---

## 2. 5 个包容维度

| 维度 | 标准 | Owner |
|---|---|---|
| **视觉**(色觉/弱视) | 对比度 ≥ 4.5:1(正文)/ 3:1(大字)/ 颜色不作为唯一信息载体 | 视觉设计师 |
| **运动**(精细操作) | 可点区 ≥ 44pt × 44pt / 长按 / 拖拽有替代方案 | 交互设计师 |
| **听觉**(听障) | 视频内容字幕 / 音频反馈有视觉冗余 | 内容设计师 |
| **认知**(老人/初次使用) | 文案动词明确 / 不用反讽 / 流程线性 | UX writer + 体验设计师 |
| **环境**(屏幕缩放/弱网) | 字号 85-150% / 弱网降级 / 离线提示 | 工程师 + 体验设计师 |

---

## 3. 反面是什么

| ❌ 反面行为 | 解释 |
|---|---|
| 仅靠颜色传达信息 | "红色提示错误"对色盲用户失效 |
| 触达区 < 44pt | 老人 / 运动场景点击失败率高 |
| 文字与背景对比 < 4.5:1 | 弱视用户看不清 |
| 弹窗只能用手势关闭 | 键盘 / VoiceOver 用户卡死 |
| 极端字号下布局崩 | 字号 150% 时按钮文字截断 |

---

## 4. 京东适老模式的设计判断

**关键判断**:适老模式**不是另一个 APP**,是标准模式的"包容性增强分支"。

| 维度 | 标准模式 | 适老模式 |
|---|---|---|
| 字号 | 默认 | 放大 130-150% |
| 信息密度 | 双列流 | 单列大卡 |
| CTA | 标准尺寸 | 加大 + 描边强化 |
| 装饰 | 正常 | 简化(去除 banner 动效)|
| 文案 | 标准动词 | 更直白("加入购物车" → "买"二字按钮)|

**避免**:做成两套割裂的 APP。组件层依然是同一套,通过主题切换实现适老变体。

## 6. 引用此原则的场景

- **任何 PR 必过的 a11y checklist**:[[../../horizontal/a11y/checklist.md]]
- **新组件出生时**:必须接入 a11y 字段(`voiceover_label` / `min_touch_target`)
- **新页面上线前**:必须跑通系统字号 85% 和 150% 两个极端
- **大促设计评审**:节庆装饰是否影响视障用户识别(动效要可关)

---

## 7. 与其他原则的张力

**与"聚焦实效"**:加大字号 / 加大触达区会降低信息密度
**协调**:适老模式作为单独主题,不影响标准模式的转化效率;a11y 基础规则(对比度 / 触达区)对所有人都有益,不需协调

---

## 8. 历史踩坑

| 时间 | 事件 | 教训 |
|---|---|---|
| 2025-Q1 | disabled 用透明度模拟 | 视障用户识别率 27% / 改用 token |
| 2025-Q2 | 错误状态仅红色描边 | 色盲用户无感知 / 加图标冗余 |
| 2025-Q3 | 大促 Banner 强制动效 | 老人用户头晕 / 加可关闭按钮 |
