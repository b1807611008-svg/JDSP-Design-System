# 图标系统 · Icon

> 京东图标采用线性主体 + 等量栅格 + ICONFONT 工程化交付。**所有图标必须从 ICONFONT 库引用,不允许业务自绘**。本文档与 [JD APP 15.0 设计语言 / 基础视觉设定 / 系统图标](https://relay.jd.com/file/design?id=1896756863949619202&page_id=44%3A1827&node_id=12327%3A45045) 同步。

> **2026-05-06 同步说明**:本次将栅格规范、笔触规范与 ICONFONT 维护流程全部对齐到 Relay 15.0;v0.9 草案中的 `12pt / 32pt / 48pt` 阶梯在 15.0 主流程中未规范,标记为待补充。

---

## 1. 设计理念(15.0 原文)

适合任何尺寸图像,有规律、平滑的设计主张,需要功能性体现的良好图像意境,以图标、连接、形状、动效、适用几个维度通融统一为设计原则。

- a. 造型简单 —— 造型简洁元素少,形式上能满足设计承载,但又不显得浪费冗余
- b. 表意准确 —— 准确传达每一个不同图标的内在含义,在最低成本的前提下,提示信息层级和操作不混杂
- c. 系统统一 —— 系统整体规范,确认设计准确
- d. 品牌专属 —— 用京东品牌的方式,在友善的中文性氛围内、保持对品牌资产的延承,提供并支持品牌特性,更天生有力

## 2. 栅格规范

### 基本尺寸 · 16 / 20 / 24DP

> **Token 命名(15.0 真值)**

| Token | 值 | 用途 |
|---|---|---|
| `icon.size.s` | 16pt | 列表项 / 表单 / 嵌入文字行内 |
| `icon.size.m` | 20pt | 工具栏 / 卡片操作 |
| `icon.size.l` | 24pt | Tab Bar / 标题栏 / 主要操作 |
| `icon.size.xs` | `> TODO: 15.0 主流程未列 12pt,沿用 v0.9 待确认` | 极小辅助 |
| `icon.size.xl` | `> TODO: 15.0 主流程未列 32pt+` | 频道入口 |
| `icon.size.xxl` | `> TODO: 15.0 主流程未列 48pt+` | 空状态 / 营销主图 |

**通用栅格**:基本字身框为 16/20/24DP,**形象建议为偶数**(便于实际开发)。

### 字身框 · 26×26DP

图标字号尺寸最低不得小于 16DP(建议 20、24);字身框对图标的容器,需在 16DP 字身框内、字身框基础之上、四角放置图标实形:**留白为 3DP 距离**。

```
┌─────────────────────────┐
│  3DP 留白               │
│   ┌───────────────┐     │
│   │               │     │
│   │   icon body   │     │
│   │   (主体形)    │     │
│   │               │     │
│   └───────────────┘     │
│                         │
└─────────────────────────┘
   字身框 26×26DP
```

### 等量原则(主体造型与字身框平衡)

> 出于图形表达的差异化和适应性,对应 ICONFONT 的细节差异性,每图相对略大、其他风格图标在大小上视觉感受趋同。

| 主体形状 | 主体 : 字身框比例 |
|---|---|
| 圆形 / 弧形 | 1 : 1(主体造型贴满字身框)|
| 横竖矩阵 | 8 : 9 |
| 圆角矩阵 | 9 : 10 |
| 三角矩阵 | 10 : 11 |

**目的**:让圆形 / 方形 / 三角形 / 四边形等图标视觉重量平衡,不会因数学尺寸相同而看起来粗细 / 大小不一。

## 3. 笔触规范

### 笔触粗细(标准描边)
图标设计粗细为按图标尺寸,粗细确认确保图标的可读性。**辅助图标的等比为 1.5DP**,其他在 16 个尺寸适用为 1DP 下:

| 图标尺寸 | 笔触粗细 |
|---|---|
| 16DP | 1DP |
| 20DP | 1.5DP |
| 24DP | 1.5DP |
| 32DP+ | `> TODO: 15.0 未规范,沿用 2DP+` |

### 弯角处理
图形线条粗细的处理,方式相对应认应过渡为 1DP+,**弯角处过渡圆角不超过 2DP**。

### 三角斜角
方形 / 三角形 / 锐角倒圆角不可超过 2DP,锐角倒圆角不可超过 2DP 内适用。

---

## 4. 视觉权重

> 主形状面积、视觉占比一致;方形 / 弧形 / 三角形按视觉填满。

- 方形:视觉填满字身框(8:9)
- 弧形 / 圆形:视觉填满字身框(1:1)
- 三角形:视觉填满字身框(10:11)
- 四边形 / 圆角矩形:视觉填满字身框(9:10)
- 锐角倒圆角不超过 2DP

---

## 5. 内部排印规范

> 图标设计上的细节为之上,图标设计为内 ICONFONT 的字身框、辅助图标、海报场,其他用 ICONFONT 不同设备的字身框,接形式同步、笔触同步、连接形态等同步。
>
> 图标内的细节、不要过于复杂、之间留白为图标线宽的 1.5 倍,以保证图标在中小尺寸下的清晰度。

- 多线时同一线宽,避免粗细混排
- 内部留白 ≈ 线宽 × 1.5,防止视觉粘连
- 图标 + 文字:**图标尺寸与字号对齐;间距与正文行间距一致**

## 6. 三种风格(沿用 v0.9)

| 风格 | 用途 | 关键场景 |
|---|---|---|
| **线性**(line) | 默认风格,80% 场景 | 工具栏 / 列表 / 表单 |
| **面性**(filled) | 强调态(选中 / 激活) | Tab Bar 选中 / 收藏已选 |
| **双色**(duotone) | 大促 / 营销 / 装饰 | 频道 icon / 优惠券 |

**配对规则**:
- 线性 + 面性 配对使用(切换状态)
- 双色不与线 / 面性混用在同一组

`> TODO: Relay 15.0 当前节点未明确「线性 / 面性 / 双色」三风格 token 化分层,沿用 v0.9 草案。需与图标组确认是否在 15.0 中保留三风格架构。`

## 7. 图标库分类(沿用 v0.9)

```
icon-library/
├── general/      # 通用(箭头 / 加减 / 关闭 / 设置)
├── action/       # 操作(分享 / 下载 / 编辑 / 删除)

├── commerce/     # 商业(购物车 / 订单 / 优惠券 / 配送)
├── social/       # 社交(收藏 / 关注 / 评论 / 点赞)
├── communication/ # 沟通(消息 / 通知 / 客服)

└── illustration/ # 大尺寸插画(空状态 / 引导)
```

## 8. 颜色

图标颜色继承父元素,**不允许图标自带色值**(除双色 / 装饰图标)。

```css
/* 默认 */
.icon { color: var(--color-neutral-text-primary); }

/* 错误状态 */
.icon-error { color: var(--color-semantic-danger); }

/* 选中态 */
.icon-active { color: var(--color-brand-primary); }
```

**双色图标例外**:在 SVG 内嵌 currentColor + 第二色,通过 props 控制。

## 9. 图标 + 文字组合

> Relay 15.0 原则:**图标尺寸与字号对齐,行间距与正文行间距一致**。

| 组合 | 图标尺寸 | 文字字号 | 间距 |
|---|---|---|---|
| 图标 + body.standard | 16pt(`icon.size.s`) | 14pt | `spacing.4`(8pt) |
| 图标 + body.standard | 20pt(`icon.size.m`) | 14pt | `spacing.4`(8pt) |
| 图标 + heading.module | 24pt(`icon.size.l`) | 15pt | `spacing.6`(12pt) |
| Tab Bar 图标 + label | 24pt(`icon.size.l`) | 10pt | `spacing.2`(4pt),垂直 |

## 10. ICONFONT 维护流程

> 大致步骤为:用户提交图标设计稿 → 上传到 ICONFONT 字库 → 由设计接单审核(校验设计资源) → 由前端实现 ICONFONT。

### 流程图

```
设计师提交图标
   ↓
上传 ICONFONT 字库
   ↓
设计组审核(校验栅格 / 笔触 / 等量原则)
   ↓
前端 ICONFONT 出码(unicode 编码)
   ↓
通过 unicode 在 APP 中调用
```

### 命名规范
- 图标名称单 unicode 编码:每个图标的最终 unicode 编码作为 ICONFONT 调用的稳定 ID
- ICONFONT 由前端整理,通过 unicode 调用
- 设计稿不应使用语义名称(如 `cart`)直接当作交付 ID,以 unicode 为准

### 样例命名清单
```
failure
warning
tips
ask
home
message
user
scan
photograph
```

(以上为设计交付时的语义占位名,前端转换为 unicode 后入库)

## 11. 反例

| ❌ 反面 | 解释 |
|---|---|
| 业务自绘图标 | 风格与库不一致 |
| 图标硬编码色值 | 不响应主题切换 |
| 不同栅格图标混用 | 视觉重量不一致 |
| 极小尺寸用极复杂图标 | 16pt 图标内放 5 个细节,看不清 |
| 内部留白 < 1.5 × 线宽 | 视觉粘连 |
| 弯角倒圆角 > 2DP | 破坏 15.0 几何特征 |
| Tab Bar 用纯线性(不切换) | 选中态视觉弱 |
