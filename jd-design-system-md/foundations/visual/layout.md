# 布局系统 · Layout

> 京东 15.0 布局 = **375×812 标准框架 + 2 类结构(平铺式 / 卡片式)+ 双梯度间距**。本文档与 [JD APP 15.0 设计语言 / 基础视觉设定 / 结构](https://relay.jd.com/file/design?id=1896756863949619202&page_id=44%3A1827&node_id=4292%3A18550) 同步。

> **2026-05-06 同步说明**:本次将草案中的 12 列栅格 / Material 风格双栏布局收敛到 Relay 15.0 实际投产的「**逻辑尺寸 375×812 + 平铺/卡片二分**」。

---

## 1. 系统框架(逻辑尺寸 375×812DP)

> Relay 原文:**以设计建议的逻辑尺寸 375×812DP 为例,划分屏幕内区域,以免影响操作体验。**

```
┌─────────────────────────┐  ← 顶部状态栏 44DP
├─────────────────────────┤
│ ⏐                     ⏐ │  ← 安全区域距屏幕左右 8DP(红色,沉浸式氛围)
│ ⏐                     ⏐ │
│ ⏐    设计区域(绿色) ⏐ │
│ ⏐                     ⏐ │
│ ⏐                     ⏐ │
├─────────────────────────┤  ← 起始指示器 34DP(蓝色,可放浏览内容,禁放操作)
└─────────────────────────┘
```

| 区域 | 高度 / 宽度 | 颜色标识 | 限制 |
|---|---|---|---|
| 顶部状态栏 | 44DP | - | 系统区域 |
| 屏幕左右安全距离 | 8DP | 红色 | **仅可放置沉浸式氛围** |
| 起始指示器 | 34DP | 蓝色 | 可放浏览内容,**禁放操作** |
| 设计区域 | 中央剩余 | 绿色 | 主要内容区域 |

**Token**:
- `spacing.safe-area.status-bar` = 44DP
- `spacing.safe-area.screen-edge` = 8DP(沉浸式)
- `spacing.safe-area.home-indicator` = 34DP

## 2. 结构二分:平铺式 vs 卡片式

> Relay 原文:**列举首页、搜索、商详、结算、订详等核心模块的框架,基于模块核心目标给予差异化结构,进行有效导流和高效展示** —— 导购入口型为卡片式,内容详情型为平铺式。

### a. 平铺式结构(内容详情型)

**适用**:商详 / 结算 / 订单详情等**单一内容型**页面
**视觉特征**:**直角拉通**,减少层级嵌套,**最大化利用空间,提供简洁平坦的视觉体验**

```
┌─────────────────┐
│  状态栏 44       │
├─────────────────┤
│  导航栏 44       │
├─────────────────┤
│                 │
│   全宽内容区    │  ← 直角,边对齐到屏幕
│                 │
│                 │
└─────────────────┘
```

**典型场景**:
- 商详(PDP):大图通栏,信息直接铺到边
- 结算页:订单 / 支付 / 收货地址直接铺
- 订单详情:物流 / 商品 / 售后

### b. 卡片式结构(导购入口型)

**适用**:首页 / 搜索 / 购物车等**入口型**页面
**视觉特征**:**小圆角卡片收拢信息,聚焦视线**

```
┌─────────────────┐
│  状态栏 44       │
├─────────────────┤
│  导航栏 44       │
├─────────────────┤
│ ┌─────────────┐ │
│ │ 卡片 1      │ │  ← 6DP 圆角,8DP 左右间距
│ └─────────────┘ │
│ ┌─────────────┐ │
│ │ 卡片 2      │ │
│ └─────────────┘ │
└─────────────────┘
```

**典型场景**:
- 首页:楼层 / Feeds 卡片
- 搜索结果:双列卡片
- 购物车:商品列表卡片

## 4. 安全区(Safe Area)

iOS / Android 都有刘海 / Dynamic Island / 凹口屏 / Home Indicator。

```
┌────────────────────┐
│  status bar / 刘海 │  ← top safe area(自动获取)44DP
├────────────────────┤
│                    │
│   设计区域          │
│                    │
├────────────────────┤
│  起始指示器 34DP    │  ← bottom safe area
└────────────────────┘
```

**铁律**:
- 任何固定底部元素(底部 CTA / Tab Bar / 浮动按钮)必须考虑 bottom safe area
- 全屏模态背景必须延伸到 safe area 外(否则刘海下露出底色)
- **起始指示器(34DP)区域可放浏览内容,但禁放操作**(用户可能误触系统手势)

## 5. 双梯度间距(对应平铺 / 卡片二分)

> 详见 [[../tokens/spacing.md]]。本节给出布局层面的速查。

### 平台型(平铺式 / 内容详情)
- 通栏间距: 12DP
- 卡片内上下: 16DP
- 卡片内左右: 12DP
- 区块内子元素: 8DP

### 导购型(卡片式 / 导购入口)
- 通栏间距: 8DP
- 卡片内上下: 12DP
- 卡片内左右: 8DP
- 区块内子元素: 6DP
- Feeds 横纵: 7DP(整除像素特殊值)

### 通用
- 非卡片承载元素左右: 16DP
- 紧密关联(图标-文字 / 文字-箭头): 4DP

## 6. 双列流标准

> 京东 APP 商品流主要形态(导购型卡片式)。

| 维度 | 取值 |
|---|---|
| 基准屏宽 | 375pt |
| 卡片宽 | (375 - 8×2 - 7) / 2 = 176pt(Feeds 7DP gap)|
| 横纵间距 | Feeds 7DP / 普通双列 8DP |
| 卡圆角 | `radius.role.card` = 6pt(对应 Relay `radius_base`) |
| 卡阴影 | none(默认) |
| 卡片节奏 | 商品图(1:1) → 标题区 → 价格区 → 角标 → 加购按钮(可选) |

## 9. 沉浸式

部分页面采用沉浸式(导航栏与内容融合),专用屏幕左右安全区域(8DP)放置沉浸式氛围。

适用:
- PDP 顶部图(状态栏文字色根据图色自动切换)
- 大促首屏 Banner
- 视频流

**实现**:导航栏背景透明 + 滚动渐变到不透明,详见 [[../interaction/navigation.md]]。

## 10. 反例

| ❌ 反面 | 解释 |
|---|---|
| 平台型页面用卡片式(商详做成卡片) | 违反 15.0 「内容详情型 = 平铺式」原则 |
| 导购型页面用平铺式(首页直接铺) | 失去信息聚合,视线散乱 |
| 固定底部 CTA 不考虑 safe area | iPhone X+ 用户被起始指示器挡住 |
| 在起始指示器(34DP)区域放操作按钮 | 触发系统手势冲突 |
| 双列卡片宽度硬编码 | 在不同屏宽下溢出 |
| 边距硬编码(如 padding: 20px) | 不引用 spacing token |
| 屏幕左右 8DP 区域放操作组件 | 该区域**仅可放置沉浸式氛围** |
| 首屏内容不考虑刘海 | 关键文字被遮挡 |
