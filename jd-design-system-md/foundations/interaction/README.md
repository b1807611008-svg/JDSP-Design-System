## 交互核心原则

| 原则 | 解释 |
|---|---|
| 反馈即时 | 任何用户操作 200ms 内必须有反馈(视觉 / 触觉 / 听觉至少一种)|
| 预期一致 | 同一手势在不同场景行为一致(下拉刷新 / 上滑切下页 / 左滑返回)|
| 撤销可达 | 危险操作必须有撤销(删除 / 取消订单 / 退订)|
| 错误可恢复 | 用户犯错后能快速回到正常状态 |

---

## 与原子组件的边界

- 原子组件(`Button` / `Input` / `Modal`)定义**行为契约**(loading 拦截点击 / 校验时机 / 蒙层点击关闭)
- 业务组件继承原子组件契约,**不允许重新定义**(如自己实现一个 disabled 不拦截点击的按钮)
- 跨业务一致性靠 foundations 层保证
