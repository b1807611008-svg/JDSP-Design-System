# 手势 · Gesture

> iOS / Android 手势体系基本通用。**不要发明新手势**。京东 APP 用户来自多端多代,新手势学习成本极高。

---

## 1. 标准手势清单

| 手势 | 用途 | 触发反馈 |
|---|---|---|
| 单击 / Tap | 触发主操作 | 视觉按下 + 触觉(选择性)|
| 双击 / Double Tap | 点赞 / 全屏切换 | 视觉强反馈 |
| 长按 / Long Press | 二级菜单 / 排序 / 引用 | 触觉 + 视觉膨胀 |
| 左滑 / Swipe Left | 返回(iOS 系统手势)/ 列表项删除 | 视觉拖拽跟随 |
| 右滑 / Swipe Right | 返回(部分场景)/ 抽屉打开 | 同上 |
| 上滑 / Swipe Up | 上一页 / 信息流加载下一条 | 视觉惯性滚动 |
| 下滑 / Swipe Down | 下拉刷新 / 关闭 sheet | 加载圈动画 |
| 双指捏合 / Pinch | 图片缩放 | 实时跟手 |
| 双指旋转 | 图片旋转(仅 PDP 主图查看)| 实时跟手 |

---

## 2. 触发区域

**最小触达区**:44pt × 44pt(WCAG 2.5.5 + iOS HIG + Material 共识)

**视觉小、触达大**:
- 视觉 24pt 图标按钮 → 实际触达 44×44 透明热区
- 关闭 X 按钮(右上 12pt)→ 实际触达 44×44

**触达拥挤区**(同时多个可点击元素):
- 元素间距 ≥ `spacing.4` (8pt)
- 关键操作单独占行,避免与其他可点击元素相邻

---

## 3. 多手势冲突

**滚动 vs 横滑列表项**:
- 默认垂直滚动,横向 30pt+ 才触发横滑
- 横滑列表项需可视化拖拽提示("拉一拉就有惊喜")

**单击 vs 双击**:
- 单击立即响应(不等双击窗口)
- 双击触发额外操作(双击点赞)
- 不允许"等 200ms 看是不是双击"的设计

---

## 4. 下拉刷新

| 阶段 | 视觉 | 触发 |
|---|---|---|
| 拉动中 | 加载图标渐显 | 用户拖拽 |
| 触发阈值 | 加载图标变实色 + 触觉 | 拖拽超过 64pt |
| 释放后 | 加载圈动画 | 自动开始刷新 |
| 完成 | 显示 Toast"已是最新" + 收回 | 自动 |

---

## 5. 左滑列表项

| 滑动距离 | 操作 |
|---|---|
| 0-25% | 显示部分操作按钮 |
| 25-75% | 完整显示操作按钮(典型:删除 / 收藏)|
| 75%+ | 触发主操作(如直接删除)|

**安全设计**:左滑触发"删除"必须有撤销 Toast,5 秒内可撤销。

---

## 6. 触觉反馈 · Haptic Feedback

| 场景 | 强度 |
|---|---|
| 按钮按下 | Light |
| 切换 Tab | Light |
| 切换状态(开关 / 单选 / 多选) | Medium |
| 加购 / 收藏成功 | Medium / Soft |
| 错误 / 警告 | Heavy / Rigid |
| 长按触发菜单 | Medium |

iOS:UIImpactFeedbackGenerator
Android:HapticFeedbackConstants

**a11y**:用户可在系统设置关闭触觉,代码不要硬编码强制触发。

---

## 7. 反例

| ❌ 反面 | 解释 |
|---|---|
| 自定义新手势(如三指滑动)| 学习成本高 |
| 单击和双击同一区域不同行为(无法区分)| 用户困惑 |
| 触达区 < 44pt 没有透明热区扩大 | 老人 / 运动场景失败率高 |
| 下拉刷新阈值过低(< 50pt)| 误触发 |
| 长按无视觉反馈(直接弹菜单)| 误触发 / 不知道发生了什么 |
