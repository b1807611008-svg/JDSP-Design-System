# 表单规范 · Form

> 表单 = 用户输入数据 + 校验 + 提交。**京东表单的难点是地址 / 身份证 / 银行卡等长串输入**——格式化、校验时机、键盘衔接都是踩坑点。

---

## 1. 表单结构

```
┌─────────────────────────┐
│ Label              *(必填)│
│ ┌─────────────────────┐ │
│ │ Input               │ │
│ └─────────────────────┘ │
│ Helper text / Error     │
└─────────────────────────┘
```

| 元素 | 字号 | 间距 |
|---|---|---|
| Label | `body.s` (13pt) | 上 16pt |
| Input | `body.m` (15pt) | 内边距 12pt |
| Helper / Error | `caption.l` (12pt) | 上 4pt |

---

## 2. Input 状态

| 状态 | 视觉 |
|---|---|
| default | `border.default` 描边 |
| focus | `border.brand` 描边 + `shadow.inner.input-focus` |
| filled | `border.default` 描边 + 文字色 primary |
| error | `border.danger` 描边 + 错误图标 + 错误 helper |
| disabled | `bg.disabled` + `text.disabled` |
| readonly | `bg.subtle` + 禁止编辑光标 |

---

## 3. 校验时机

| 时机 | 校验 | 用途 |
|---|---|---|
| 输入中 | 仅格式化(如手机号自动加空格)| 不打扰 |
| 失去焦点 | 完整校验该字段 | 标准时机 |
| 提交时 | 全表单校验 | 兜底 |
| 实时(慎用)| 长字段(密码强度) | 实时反馈强度 |

**铁律**:
- **不要边输入边显示错误**(用户还没输完就报错,体验差)
- **失去焦点 + 提交时**两个时机做完整校验

---

## 4. 错误提示

| 位置 | 何时用 |
|---|---|
| Input 下方 helper | 单字段错误 |
| Input 内右侧图标 | 配合 helper,增强识别 |
| 顶部 banner | 表单级错误(如"网络异常,请重试")|
| Toast | 提交时的网络/服务错误 |

错误文案规则:
- 具体说明问题("手机号格式错误"而非"输入错误")
- 给出修复建议("请输入 11 位手机号")
- 不指责用户("您的输入有误")

---

## 5. 键盘衔接

| 字段类型 | 键盘类型 |
|---|---|
| 普通文字 | text |
| 数字(数量 / 价格) | number / decimal |
| 手机号 | phone |
| 邮箱 | email |
| 身份证 | text(自定义) |
| 验证码 | number |
| 密码 | password(隐藏)|
| URL | url |
| 搜索 | search |

**铁律**:
- 切换字段时**不应关闭键盘**(用户感觉跳动)
- 键盘上方有"完成"按钮(iOS)/ Tab 切换(Android)
- 键盘弹出时不遮挡当前 input(自动滚动)

---

## 6. 自动格式化

| 字段 | 格式化 |
|---|---|
| 手机号 | 138 1234 5678(每 3-4 位空格) |
| 身份证 | 110101 1990 0101 1234(分段)|
| 银行卡 | 6228 4800 1234 5678(每 4 位空格)|
| 金额 | ¥1,234.56(千分位 + 2 位小数)|
| 日期 | 2026-04-29(自动加 "-")|

格式化字符**不计入校验长度**,提交时去除空格 / 横线。

---

## 7. 表单提交

| 阶段 | 视觉 |
|---|---|
| 提交按钮按下 | 按钮变 loading 状态 |
| 提交中 | 按钮 loading + 屏蔽点击 |
| 成功 | Toast / 跳转 |
| 失败 | 按钮恢复 + 错误提示 |

**关键**:loading 状态必须屏蔽点击(详见 [[../components-base/action/button/donts.md#LOADING_NOT_BLOCKING]]),防止重复提交。

---

## 8. 长表单

| 长度 | 处理 |
|---|---|
| < 10 字段 | 单页表单 |
| 10-20 字段 | 分组(用分隔线 / 卡片) |
| 20+ 字段 | 分步表单(进度条 + 上一步 / 下一步)|

**铁律**:**长表单必须有进度感知**。用户不能感觉"无限填下去"。

---

## 9. 反例

| ❌ 反面 | 解释 |
|---|---|
| 边输入边报错 | 用户还没填完 |
| 错误提示模糊("输入有误") | 不知道怎么改 |
| 切换字段键盘消失再弹 | 视觉跳动 |
| 提交按钮 loading 不拦截点击 | 重复提交 |
| 必填项不标 `*` | 用户漏填 |
