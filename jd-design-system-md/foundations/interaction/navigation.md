# 导航规范 · Navigation

> 京东 APP 导航包含 Tab Bar / 顶部导航 / 路由层级 / 返回机制。**Tab Bar 是用户的"地图"**——结构稳定 = 用户安全感。

---

## 1. Tab Bar(底部主导航)

### 标准 5 Tab
| 顺序 | Tab | 图标(线/面)| 入口 |
|---|---|---|---|
| 1 | 首页 | 房屋 | 推荐流 |
| 2 | 频道(分类)| 网格 | 商品分类 |
| 3 | 发现(种草 / 短视频)| 圆框 | 内容流 |
| 4 | 购物车 | 购物车 | 购物车 |
| 5 | 我的 | 头像 | 个人中心 |

### Tab Bar 规则
- 高度:49pt(iOS)/ 48pt(Android),+ bottom safe area
- 选中态:面性图标 + 京东红 + 文字加粗
- 未选中态:线性图标 + `text.tertiary` + 常规字重
- 角标:右上 dot 或数字(数字 ≤ 99,超出显示 "99+")
- **不允许临时增减 Tab**(改变结构 = 用户重新建立心智模型)

### 文案
- ≤ 4 字(中文)/ ≤ 8 字符(英文)
- 不允许临时改文案(影响用户记忆)

## 2. 顶部导航栏

### 标准结构
```
[← 返回] [页面标题(居中)] [操作按钮(右)]
```

### 规则
- 高度:44pt
- 返回按钮:左侧,touchable area 44×44
- 标题:`heading.h2` (20pt) semibold,居中
- 右侧操作:1-2 个图标按钮(更多操作放"⋯"菜单)
- 沉浸式场景:导航栏背景透明,滚动渐变到不透明

### iOS / Android 差异
| 维度 | iOS | Android |
|---|---|---|
| 标题位置 | 居中 | 左对齐(可配置)|
| 返回按钮 | 文字"返回" + 箭头 | 仅箭头 |
| 系统手势返回 | 左滑边缘 | 无(用 Android 自带返回键) |

详见 [[../../horizontal/multi-platform/ios-android-diff.md]]

## 3. 路由层级

```
首页(Tab 1)
├─ 频道页 → 商品列表 → PDP → SKU 选择 → 加购/购物车
├─ 频道(Tab 2)
├─ 发现(Tab 3)
├─ 购物车(Tab 4)
│   └─ 提交订单 → 选择地址 → 结算 → 支付 → 订单详情
└─ 我的(Tab 4)
    ├─ 订单
    ├─ 优惠券
    ├─ 设置 → 二级 → 三级
    └─ ...
```

**层级铁律**:**单一流程深度 ≤ 5 层**。超过 5 层用户迷路率高。

---

## 4. 返回机制

| 场景 | 行为 |
|---|---|
| 二级页面 → 一级 | 系统返回(箭头 / 左滑 / Android 返回键)|
| 多级深入 → 主页 | 提供"返回首页"快捷入口 |
| 模态 → 关闭 | 关闭 X / 蒙层点击 / 下滑 sheet |
| 表单未完成 → 返回 | 提示"放弃编辑?"二次确认 |
| 支付中 → 返回 | 强提示"是否放弃支付" |

## 5. 沉浸式 / 透明导航

部分场景使用沉浸式:

| 场景 | 行为 |
|---|---|
| PDP 顶部商品图 | 图占满顶部,导航透明,滚动时渐变到白底 |
| 大促首屏 | 导航透明,Banner 占满 |
| 视频流 | 全屏沉浸,半透明导航 |

**实现细节**:
- 状态栏文字色根据顶部背景色自动切换
- 导航栏滚动到 64pt 后开始渐变,128pt 全实色
- 沉浸式状态下返回按钮带半透明圆形背景(增强可见性)

## 6. 锚链接 / 楼层切换

长页面(频道页 / PDP)有锚点导航:

```
顶部 Tab:[全部] [上衣] [裤装] [鞋子] [配饰]
       ↓ 点击
     页面滚动到对应楼层 + Tab 跟随选中
```

- 锚链接 Tab 高度 40pt
- 滚动惯性 + 加速度匹配 `motion.duration.fast`
- 用户手动滚动时锚 Tab 自动跟随

---

## 7. 反例

| ❌ 反面 | 解释 |
|---|---|
| 临时改 Tab Bar 结构 | 用户重新建立心智模型 |
| 同一手势在不同页面行为不一致 | 学习成本高 |
| 单一流程深度 > 5 层 | 用户迷路 |
| 返回按钮位置漂移 | 用户找不到 |
| 沉浸式不切换状态栏色 | 文字看不见 |
