# 通知规范 · Notification

> 系统通知 / 应用内通知 / 角标 / 消息中心。**通知是双刃剑**——用得好建立信任,用得糟用户关推送。

---

## 1. 通知层级

| 层级 | 触达 | 用户 opt-in |
|---|---|---|
| 系统级 push 通知 | 锁屏 / 通知栏 | 用户授权 |
| 应用内 banner | APP 内顶部下拉 | 自动(在 APP 内时)|
| 应用内浮层 / 弹窗 | APP 内 | 自动 |
| 角标 / dot | Tab Bar / 模块入口 | 自动 |
| 消息中心 | 我的 → 消息 | 自动 |

---

## 2. 系统级 Push 通知

### 类型
| 类别 | 频次上限 | 用户可关 |
|---|---|---|
| 订单状态(发货/到货/收货)| 业务触发 | 否 |
| 支付提醒(待支付)| 业务触发 | 是 |
| 营销通知(秒杀 / 优惠)| ≤ 3 次/天 | 是 |
| 系统通知(更新 / 公告)| ≤ 1 次/周 | 否 |
| 客服回复 | 业务触发 | 否 |

**铁律**:
- 营销通知必须用户 opt-in,默认关闭(2024+ 监管要求)
- 通知频次过高自动降权(系统层面)
- 通知内容要可消化(标题 + 摘要,点击进入详情)

### 通知文案规则
- 标题 ≤ 20 字
- 内容 ≤ 50 字
- 不用全大写 / 过多感叹号
- 避免诱导点击("点这里有惊喜!" 是反例)

---

## 3. 应用内 Banner

APP 在前台时收到推送 → 顶部下拉 banner:

| 元素 | 描述 |
|---|---|
| 图标 | App icon 或场景图标 |
| 标题 | `body.m` semibold,1 行 |
| 内容 | `body.s` regular,1 行,溢出省略 |
| 关闭 X | 右上角 |
| 触感 | Light haptic |
| 时长 | 5 秒(可上滑提前关闭)|

---

## 4. 应用内浮层 / 弹窗

| 类型 | 用途 |
|---|---|
| 引导浮层 | 首次进入新功能引导 |
| 营销浮层 | 优惠券领取 / 活动入口 |
| 系统提示 | 强制更新 / 隐私协议 |

**铁律**:
- 营销浮层每用户每天 ≤ 1 次
- 必须可关闭(右上 X 或点击空白)
- 引导浮层不超过 3 步

---

## 5. 角标 · Badge

### 类型
| Token | 用途 |
|---|---|
| `badge.dot` | 红点(2pt 圆点),仅表示"有新内容" |
| `badge.number` | 数字(1-99,超出"99+")|
| `badge.text` | 文字(NEW / 热)|

### 位置
- Tab Bar 上 / 入口图标右上 / 头像右上
- 偏移:右移 4pt + 上移 4pt(避免遮挡图标)

### 规则
- dot 直径:6pt
- 数字 / 文字:`caption.m` (11pt)+ 京东红底 + 白字 + `radius.full`
- 红点比数字优先级低(数字"5" > dot)
- **不滥用**:每个 Tab / 入口最多 1 个角标

---

## 6. 消息中心

我的 → 消息 → 列表展示所有通知。

### 分类
- 全部
- 订单消息
- 优惠活动
- 互动消息(评论 / 点赞)
- 系统通知

### 列表项
```
[图标] 标题(粗) ........ 时间
       内容摘要(2 行内)
       [操作按钮(可选)]
```

- 未读 dot:左边 2pt 红点
- 已读:文字色降为 `text.secondary`
- 长按:删除 / 标记已读

---

## 7. 反例

| ❌ 反面 | 解释 |
|---|---|
| 营销通知频次过高(每天 5+) | 用户关推送 |
| 通知文案诱导("点这里有惊喜")| 监管违规 |
| 角标乱用(每个图标都有 dot) | 视觉污染 |
| 消息中心无法清空 / 标记已读 | 用户压力大 |
| Banner 不可关闭 | 阻塞用户 |
