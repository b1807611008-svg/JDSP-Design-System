# 模态规范 · Modal

> 京东 APP 模态体系:**Toast / Dialog / Sheet / 全屏蒙层** 4 种,各有边界。**层级冲突是最常见的事故源**。

---

## 1. 4 种模态对比

| 类型 | 用途 | 阻塞性 | 自动关闭 |
|---|---|---|---|
| **Toast** | 操作反馈("加购成功" / "已删除")| 非阻塞 | 是(2-3 秒)|
| **Dialog** | 关键决策(确认删除 / 隐私同意)| 完全阻塞 | 否 |
| **Sheet**(底部抽屉)| SKU 选择 / 优惠券领取 / 二级筛选 | 半阻塞 | 否(用户主动关闭)|
| **Lightbox**(全屏蒙层)| 图片大图 / 视频全屏 / 引导浮层 | 完全阻塞 | 否 |

---

## 2. Toast

### 类型
| Token | 用途 |
|---|---|
| `toast.success` | 成功反馈("加购成功") |
| `toast.warning` | 警告("库存紧张") |
| `toast.error` | 错误("网络异常") |
| `toast.info` | 普通提示("已收藏") |
| `toast.loading` | 加载中(配合 spinner)|

### 规则
- 位置:屏幕中央 / 顶部(根据业务,统一一种)
- 时长:2 秒(默认)/ 3 秒(长文)/ 1.5 秒(短反馈)
- 同时只能存在 1 个 Toast(新 Toast 替换旧)
- 不阻塞用户操作
- 不允许 Toast 内嵌按钮(关键操作用 Dialog)

---

## 3. Dialog

### 类型
| 名称 | 用途 |
|---|---|
| Confirm | 确认型(确认删除 / 退出登录)|
| Alert | 通知型(系统消息 / 重大变更)|
| Privacy | 隐私 / 协议同意 |
| Update | 强制更新 |

### 结构
```
┌─────────────────────────┐
│      标题(可选)         │
│                         │
│      内容描述            │
│                         │
│  [取消]      [确认]     │
└─────────────────────────┘
```

### 规则
- 宽度:屏宽 80%(手机)/ 480pt(iPad)
- 圆角:`radius.modal` (16pt)
- 标题:`heading.h3`(可选)
- 内容:`body.m`,左对齐
- 按钮:右下,主按钮在右(iOS)/ 主按钮在底(Android)
- 完全阻塞,蒙层不可点击关闭(必须显式选择)
- 危险操作:取消按钮在左,危险确认按钮在右(`type=danger`)

### 文案规则
- 标题 ≤ 12 字
- 内容 ≤ 50 字(2 行)
- 按钮文案具体动词("永久删除" / "保留",不用"是" / "否")

---

## 4. Sheet(底部抽屉)

### 类型
| 名称 | 用途 |
|---|---|
| ActionSheet | 操作菜单(分享 / 举报)|
| FormSheet | 表单(SKU 选择 / 二级筛选)|
| InfoSheet | 详细信息(商品详情子页) |

### 规则
- 高度:30%-90% 屏高(可拖拽调整)
- 顶部有 4pt × 36pt 拖拽条
- 顶部圆角:`radius.16`(仅顶部圆,底部直)
- 关闭方式:下滑 / 点击蒙层 / 关闭 X 按钮
- 多级 Sheet:支持多层叠加(SKU → 优惠券 → ...)但 ≤ 3 层

### Sheet 内的关键操作
- 主 CTA 固定在 Sheet 底部(撑满宽,考虑 safe area)
- 主 CTA 上方留 8pt 间距,与内容区分

---

## 5. 蒙层 · Scrim

所有模态共享蒙层规则:

| Token | 值 | 用途 |
|---|---|---|
| `color.modal.scrim` | `rgba(0,0,0,0.45)` light / `rgba(0,0,0,0.65)` dark | 标准蒙层 |
| `color.modal.scrim.light` | `rgba(0,0,0,0.30)` | 半透明蒙层(下方仍可见)|

蒙层动效:
- 进入:`motion.fast` + `easing.decelerate` 透明度 0 → 1
- 退出:`motion.fast` + `easing.accelerate` 1 → 0

---

## 6. 模态层级冲突

**铁律**:**同一时间最多 2 层模态**(如 Sheet + Dialog 二次确认)。超过 2 层用户迷失。

层级管理:
- ToastLayer(z-index 最高,不阻塞)
- DialogLayer(完全阻塞)
- SheetLayer(半阻塞)
- LightboxLayer(完全阻塞)

后弹的覆盖先弹的,关闭后逆序回到原状态。

---

## 7. 反例

| ❌ 反面 | 解释 |
|---|---|
| Toast 内放按钮 | Toast 是一过性的,不该承载关键操作 |
| Dialog 蒙层点击关闭(可绕过决策)| Dialog 必须显式选择 |
| Sheet 内嵌 Sheet 嵌 Sheet 嵌 Sheet | 用户迷失 |
| 模态没有蒙层 | 视觉不阻塞,用户分不清主次 |
| 全屏 Lightbox 没有关闭按钮 | 用户出不去 |
| 模态进入和退出动效不一致 | 节奏混乱 |
