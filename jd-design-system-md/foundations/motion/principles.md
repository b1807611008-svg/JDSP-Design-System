# 动效设计原则

> Relay 15.0 原文:**动效设计遵循简单、统一的原则,即非必要不增加,且参数必须遵循此规范定义。**

---

## 五个典型应用场景

15.0 列举需要动效表达的典型要素:

| # | 场景 | 目的 |
|---|---|---|
| a | **降低理解成本** | 帮助用户了解当前状态(开关 / Tab 切换状态变化)|
| b | **引导视觉焦点** | 吸引用户注意力(吐司、新功能引导) |
| c | **提升流畅度** | 避免大反差变化导致的卡顿、突兀(转场 / 入出场) |
| d | **清晰层级结构** | 理解当前信息结构的全貌(半弹层、弹窗) |
| e | **运营场景下增加活力趣味** | 大促、营销气泡、强交互 Icon |

> **判断标准**:动效如果不能映射到 a-e 中任意一项,**应该删掉**。

---

## 核心原则

### 1. 非必要不增加
京东不是动效驱动的产品。**主流程(PDP / 购物车 / 结算 / 支付)动效尽量少 / 短**:
- 主流程: 优先 `motion.duration.s` (150ms)
- 一般功能页: 最多 `motion.duration.m` (200ms)
- 大促 / 营销: 可使用 `motion.duration.l` (300ms) + `easing.spring`
- 工具型页面: 极简,只保留状态切换(`motion.role.toggle`)

### 2. 参数遵循规范
**时长** 必须 ∈ {150, 200, 300}ms;**缓动** 必须 ∈ {standard, decelerate, accelerate, spring}。
任何裸贝塞尔 / 裸毫秒数都需要在 PR 阶段被 lint 拦截。

### 3. 出场降一档
进场对应尺寸 → 出场降低一个时间梯度(进 200 出 150 / 进 300 出 200)。这是 15.0 的视觉时序铁律,与「视觉关注分配」直接相关。

### 4. 减少动效响应
所有非必需动效必须能在系统「减少动效」开启时退化到 `0ms` 或纯透明度过渡。

详见 [[../../horizontal/a11y/motion-reduce.md]]。

### 5. 性能优先
- 60 FPS 是底线
- 列表滚动中不触发额外动效
- 大量元素同时动画使用 GPU 优化属性(transform / opacity)

---

## 张力与取舍

| 张力 | 决策建议 |
|---|---|
| **克制 vs 情感表达** | 主流程克制(s/m + standard);大促 / 营销可使用 spring + l |
| **物理感 vs 性能** | 移动端用预设贝塞尔 token(GPU 友好),不自定义 |
| **进场长 vs 出场短** | 严格遵守「出场降一档」,不在某场景例外 |

---

## 引用此原则的场景

- 评审带动效的方案 → 走「五个典型应用场景」+「核心原则 5 条」checklist
- 大促设计 → 在「克制」与「情感表达」之间找平衡
- 性能优化 → 物理感 vs 60 FPS 的取舍
