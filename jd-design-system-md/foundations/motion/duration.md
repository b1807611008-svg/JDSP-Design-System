# 动效时长 · Duration

> Token 见 [[../tokens/motion.md#2-时长--duration]]。本文讲"什么场景用多长时间"。

---

## Relay 15.0 的核心规则

**3 阶时长 + 1 个边界值**,按对象**尺寸**选择:

| 尺寸 | Token | 值 |
|---|---|---|
| 小 | `motion.duration.s` | **150ms** |
| 中 | `motion.duration.m` | **200ms** |
| 大 | `motion.duration.l` | **300ms** |
| (边界,可感知最短) | `motion.duration.instant` | 100ms |

**100ms 是人脑可感知的最短动效**,< 100ms 难以识别,> 1000ms 有明显迟滞感,**最佳区间 100-500ms**。引用自 Material Design + Windows 官方动效指南。

---

## 决策表(基于 Relay 15.0 实例)

| 场景 | 尺寸 | 时长 Token | 值 |
|---|---|---|---|
| 选择器 / 开关 / Tab 切换 | 小 | `s` | 150ms |
| 吐司提示入场 | 中 | `m` | 200ms |
| 吐司提示出场 | 小 | `s` | 150ms |
| 弹窗(系统型)入场 | 中 | `m` | 200ms |
| 弹窗(系统型)出场 | 小 | `s` | 150ms |
| 半弹层入场 | 大 | `l` | 300ms |
| 半弹层出场 | 中 | `m` | 200ms |
| 容器继承(从属转场)正反向 | 大 | `l` | 300ms |
| 递进上浮 进场 | 大 | `l` | 300ms |
| 递进上浮 出场 | 中 | `m` | 200ms |
| 推屏跳转 进 / 出 | 大 | `l` | 300ms |

---

## 出场降一档铁律

> 进场遵循尺寸对应时间,**出场降低一个时间梯度**。

```
进场 l (300ms) → 出场 m (200ms)
进场 m (200ms) → 出场 s (150ms)
进场 s (150ms) → 出场 s (150ms,已是最短,不再降)
```

**目的**:
- 进场: 用户关注入场元素,适当延长便于识别
- 出场: 快速离开,提升交互效率

---

## 已废弃 / 待迁移

v0.9 草案中以下时长在 15.0 主流程**不规范**,迁移路径:

| v0.9 token | 值 | 15.0 处置 |
|---|---|---|
| `instant` (0ms) | 0 | 保留为「立即,无动效」语义,等同减少动效 |
| `immediate` (100ms) | 100 | → `instant (100ms,可感知边界)` 或合并到 `s` |
| `fast` (200ms) | 200 | → `m` |
| `normal` (300ms) | 300 | → `l` |
| `slow` (500ms) | 500 | **deprecated**,无 15.0 对应,大幅动效需走治理 |
| `slower` (800ms) | 800 | **deprecated**,引导动画走单独主题包 |

## 反例

| ❌ 反面 | 解释 |
|---|---|
| 主流程动效 > 300ms | 影响转化效率,15.0 主流程上限 200ms |
| 弹窗入场 < 200ms | 跳变,无层级感 |
| 弹窗出场 = 入场时长 | 违反「出场降一档」铁律 |
| Toast 显示 < 1.5 秒 | 用户没看清就消失(显示时长另算,非动效时长) |
| 自定义时长(如 250ms) | 不在 token 表内,lint 应拦 |
