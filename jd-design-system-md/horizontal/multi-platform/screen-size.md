# 屏幕尺寸适配 · Screen Size

> 京东 APP 覆盖从 iPhone SE(320pt 宽)到 iPad Pro(1366pt 宽)的所有屏幕。**不锁死像素,自适应为本**。

---

## 1. 设备尺寸矩阵

| 设备 | 屏宽(pt) | 屏高 | 设计稿基准 |
|---|---|---|---|
| iPhone SE / iPhone mini | 320-375 | 568-812 | 375pt |
| iPhone 标准 | 375-393 | 812-852 | 375pt |
| iPhone Plus / Max | 414-430 | 896-932 | 自适应 |
| Galaxy 标准 | 360-384 | 800-892 | 375pt |
| Galaxy Plus | 412-440 | 916-1080 | 自适应 |
| 折叠屏 闭合 | 375-400 | 880+ | 375pt |
| 折叠屏 展开 | 750-820 | 880+ | 757pt |
| iPad mini | 744 | 1133 | 768pt |
| iPad 标准 | 820 | 1180 | 820pt |
| iPad Pro 11 | 834 | 1194 | 834pt |
| iPad Pro 12.9 | 1024-1366 | 1366 | 1024pt |

---

## 2. 适配策略

### 单栏布局(< 600pt 宽)
- 所有手机 + 折叠屏闭合
- 标准 12pt 边距 + 单列流 / 双列流

### 双栏布局(600-900pt 宽)
- iPad mini / iPad 标准 / 折叠屏展开
- 主栏 + 副栏(70:30)
- 商品流可改三列

### 多栏布局(> 900pt 宽)
- iPad Pro
- 主栏 + 副栏(60:40)
- 商品流四列

---

## 3. 双列流卡宽计算

```
卡宽 = (屏宽 - 边距×2 - 卡间距) / 2

iPhone SE 375pt:
卡宽 = (375 - 24 - 12) / 2 = 169.5pt

iPhone Pro Max 430pt:
卡宽 = (430 - 24 - 12) / 2 = 197pt

折叠屏 757pt(展开):
卡宽 = (757 - 32 - 12) / 3 = 237.67pt(三列)
```

---

## 4. 安全区(Safe Area)

| 区域 | 自动获取 |
|---|---|
| 顶部状态栏 | `safeAreaInsets.top` |
| 底部 Home Indicator | `safeAreaInsets.bottom` |
| 左 / 右 凹槽(横屏)| `safeAreaInsets.left/right` |

**铁律**:固定底部 / 固定顶部元素必须考虑 safe area。

---

## 5. 折叠屏特殊场景

| 场景 | 处理 |
|---|---|
| 折叠态 | 同手机 |
| 展开态 | 双栏布局或自适应放大 |
| 折-展开切换 | 自适应过渡(不卡顿)|
| 主屏 / 外屏切换 | 状态保持 |

---

## 6. 横屏

主流程页(PDP / Cart / Checkout)**不支持横屏**(京东选择简化):
- 锁定竖屏(`orientation: portrait`)
- 视频播放页例外(支持横屏)

---

## 7. 性能预算

| 屏幕 | FPS 底线 | 内存占用 |
|---|---|---|
| 旧款 iPhone SE / 老 Android | 30 FPS 可接受 | < 200MB |
| 标准 | 60 FPS | < 300MB |
| Pro / 高端 | 60 FPS(120 FPS 支持)| < 500MB |
| iPad Pro | 60 FPS | < 800MB |

---

## 8. 反例

| ❌ 反面 | 解释 |
|---|---|
| 卡宽硬编码 170pt | 在不同屏宽下溢出 |
| iPad 强制单栏(浪费空间)| 应该多栏 |
| 横屏支持但布局不适配 | 视觉错位 |
| 不考虑 safe area | iPhone X+ 用户被挡 |
| 弱机型 FPS < 20 | 卡顿严重 |
