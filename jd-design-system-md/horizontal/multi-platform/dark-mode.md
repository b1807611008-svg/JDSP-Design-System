# 深色模式 · Dark Mode

> 通过 Token 自动映射,**设计师不需要为深色模式单独设计**。但有几个特殊场景需要手工调整。

---

## 1. 实现机制

```
color.brand.primary
  ├── light: #fa2c19
  └── dark:  #ff4538
```

每个 color token 自带 light/dark 双值,系统根据当前主题自动取对应值。

详见 [[../../foundations/tokens/color.md#浅色--深色双模式]]

---

## 2. 用户控制

| 模式 | 行为 |
|---|---|
| 浅色 | 强制浅色 |
| 深色 | 强制深色 |
| 跟随系统 | 跟随 iOS / Android 系统设置 |

**默认**:跟随系统。

---

## 3. 深色模式色彩调整

| 类型 | 调整 |
|---|---|
| 文本主色 | 浅色 → 深色背景上的浅色 |
| 背景 | 白 → `#0a0a0a` 或 `#1c1c1e`(分层)|
| 描边 | 浅灰 → 深灰 |
| 阴影 | 加深 / 加宽(否则看不到层级)|
| 品牌色 | 略偏亮(如 `#ff4538` 而非 `#fa2c19`)|

---

## 4. 特殊场景需手工调整

### 渐变 / 大促装饰
深色模式下饱和度需手工降低 10-15%:
```
default: linear-gradient(135deg, #fa2c19, #ff7800)
dark:    linear-gradient(135deg, #d92516, #d96900)
```

### 大块京东红区域
深色模式下品牌色更亮(对比度提升)。

### 商品图
商品图本身不调整(白底商品图在深色模式下仍然白底,这是商品本身的属性)。
- 商品图周围的卡片背景:深色
- 商品图本身:不变

### 大促主题包
每个大促主题包都需要 light + dark 两个版本。

---

## 5. 状态栏文字色

深色模式下状态栏文字色自动切换为白色。沉浸式场景例外(根据顶部图色判断)。

---

## 6. 测试要求

每个组件 / 页面上线前必须:
- [ ] 浅色模式截图
- [ ] 深色模式截图
- [ ] 浅色 → 深色切换无视觉抖动
- [ ] 关键文字对比度在两种模式下都达标

---

## 7. 反例

| ❌ 反面 | 解释 |
|---|---|
| 不支持深色模式(强制浅色)| 系统切换时白屏闪烁 |
| 硬编码色值 | 深色模式失效 |
| 阴影不调整 | 深色背景上看不到层级 |
| 渐变不调整 | 深色模式下过亮 |
| 商品图被强制反色 | 影响商品识别 |
