## 一、L1 大规范（9 条跨家族规约）

> 所有双列卡都必须满足，无差别。源于 22 个基线的横向归纳。

### L1-1 · 左上角 = 卡片属性表达位

表达「**非默认媒介属性**」。

| 属性 | 左上角内容 |
| --- | --- |
| 视频（非默认）| 视频角标（白色播放 ▶，部分场景带时长）|
| 直播（非默认）| 红色直播图标 + 观看数 |
| 图文（默认态）| **大多数场景为空**；个人页场景显式标图文 icon |

**附加规则**：可共生话题标签（"#XXX >"）/ 浏览历史提示（"刚刚看过"）/ 用户状态（"我的关注"）。

### L1-2 · 右上角 = 辅助信号位（可为空）

| 卡型 | 右上角内容 |
| --- | --- |
| 商品直播卡 | 店铺胶囊 |
| 内容直播卡·红包雨变体 | 活动胶囊（红包雨 / 抽奖）|
| 其他卡型 | 大多为空 |

### L1-3 · 标题下方 = 关于「内容」的元信息位

视觉权重 ∝ 信号公共性：

| 信号类型 | 案例 | 视觉权重 |
| --- | --- | --- |
| 公共社交证明 | "12.3 万人种草" / "服饰美妆排行榜第 1 名" | 中-强 |
| 用户状态（个人化）| "你预约的" / "你关注的" | **弱**（灰色小字）|
| 紧凑数字（同行右侧）| "8.5 万"（观看量）| 弱-中 |

**设计哲学**：公共信号要被所有人看到（强视觉），个人化信号只对你有意义（弱视觉，记忆唤起即可）。

### L1-4 · 商品挂载浮条位置 = 反映叙事方向

| 浮条位置 | 案例 | 叙事方向 |
| --- | --- | --- |
| 封面下、标题上（灰底）| 种草卡 | 商品是**起点**（商品 → 内容是商品的延伸）|
| 标题下、最底部（白底）| 新品测评视频卡 / 明星同款卡 | 商品是**终点**（内容 → 决定是否点商品）|

### L1-5 · 底部不变量 = 家族 DNA 锚定 ⭐ 最核心规约

> 「**家族身份由底部决定，不由封面决定**」
>
> 封面可以任意变化（基础款 / 梯度叠加 / 沉浸式场景），底部不变 → 家族不变。

**实证**：本轮采集中「内容·达人沉浸式」和「商品·促销沉浸式」两张卡封面相似度极高，但底部不同 → 家族完全不同。**用户 0.5 秒扫视时锚点是底部，不是封面**。

### L1-6 · 内容卡右下互动信号位可切换

| 子样式 | 右下图标 | 用例 |
| --- | --- | --- |
| 点赞互动（基础）| ❤️ | 偏社交内容 |
| 观看量互动 | 👁 | 偏视频 / 消费内容 |
| 评论互动 | 💬 | 偏深度讨论 |
| 收藏互动 | ⭐ | 偏工具 / 收藏型 |

**骨架固定，信号可选**——业务线表达自由度在「右下信号位」。

### L1-7 · 封面浮层 = 决策信息前置位（描述性）

| 浮层内容类型 | 典型案例 |
| --- | --- |
| 销量证明 | "本场热卖 ×286" |
| 模拟弹幕 | "星火·天冷，求保暖" |
| 关键属性标签 | "麂皮面料" |
| 沉浸式商品卡 | 白底商品缩略 + 价格 |
| 活动叠字 | "618 采销降价火热进行中" |

**视觉权重要克制不能完全遮盖封面**。具体规则由业务方决定，Skill 不下钻。

### L1-8 · 梯度叠加封面 = 一种封面结构类型（描述性）

封面是多层图层叠加（场景图 → 白底图 → 直播间封面合成）。Skill 识别「这是梯度叠加」即可，不深挖每层。

### L1-9 · 昵称旁 = 关于「人」的元信息位 ★ v0.4.1 新增

承载用户认证身份等关于**人本身**的元信息。位置：**紧贴昵称右侧**。

| 内容类型 | 案例 | 视觉风格 |
| --- | --- | --- |
| 用户认证身份 | "求真官"（点评业务）| 红色小胶囊 |
| 达人 / 品牌官 / V 标 | 待采样确认 | — |

## 二、设计哲学：元信息按「主语」分位置

> 这条是 L1-3 + L1-9 共同支撑的**元规则**，写出来作为反例的判断依据：

> **元信息按主语分位置，位置 = 主语的视觉锚点**
>
> - 陈述**内容**（这内容怎么样）→ **标题下方**（紧贴内容描述）
> - 陈述**人**（这人是谁）→ **昵称旁**（紧贴人）

用户想了解「谁说的」，眼睛会自然落到说话人附近；想了解「内容怎么样」，眼睛会落到内容附近。**位置不是布局选择，是用户视线路径的设计**。

## 三、L2 家族专属规则

L2 = 按 `card_type` 的家族再走对应规则。

| 家族 | L2 规则源 | 关键差异点 |
| --- | --- | --- |
| 内容 | Skill 仓库 `component.md` 第 2/3/4/6 节 | 头像必选 / 互动数据右下必选 / 直播卡可叠加红包雨/超头/沉浸式变体 |
| 点评 | `component.md` 基础 + `business-baselines.md` §3.4 | 与内容家族骨架同 + 标题下方种草数（公共强信号）+ 昵称旁可挂身份胶囊 |
| 圈子 | `business-baselines.md` §3.3 | **无作者、无互动**（最特殊）；底部只有圈子图标 + 圈子名 |
| 商品 | ⚠️ **L2 待补**（当前只走 L1）| — |
| 新品 | `business-baselines.md` §3.5 | 按子类型分（入口 / 直播入口 / 测评 / 趋势 / 明星同款）|

## 四、视觉规约的边界（什么不在 Skill 观察范围）

| 不观察 | 理由 |
| --- | --- |
| 封面图具体内容 / 构图 / 比例偏向 | 用户 / 运营上传，设计师无法强控制 |
| 标题具体文案 | 只看字数合规性，不评价写得好不好 |
| 头像具体图像 | 用户上传 |
| 运营配图色彩倾向 | 内容层 |
| **图片比例超出规范值域**（如 1:2 极端比例） | ⚠️ 这种**要观察**——属规范违规 |

详见 [`donts.md`](./donts.md) 的「原则 7：结构优先、内容无涉」。

视觉规约的具体数值应引用 Zone 2 foundations 的 token：

| 规约维度 | 引用 token |
| --- | --- |
