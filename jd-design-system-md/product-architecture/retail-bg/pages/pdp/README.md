# 商品详情页 · PDP (Product Detail Page)

> 京东 APP 转化漏斗的关键节点。**PDP 设计的核心矛盾是"信息全 vs 决策快"**——既要展示完整商品信息,又要让用户快速加购。

---

## 模块清单(从上到下)

```
PDP
├── [Module] 顶部沉浸式导航(滚动渐变)
├── [Module] 商品图区(主图 + 缩略图 + 视频)
├── [Module] 价格 + 促销标识(主价 / 划线价 / 限时 / 优惠券领取)
├── [Module] 商品标题 + 简短卖点
├── [Module] 规格选择(SKU - 颜色 / 尺码)
├── [Module] 配送地址 + 配送时效
├── [Module] 服务承诺(7 天无理由 / 自营 / 京东保)
├── [Module] 评价摘要(评分 + 标签 + 精选评价)
├── [Module] 详情(图文滚动展示)
├── [Module] 推荐相关 / 相似商品
└── [Bottom Bar] 加入购物车 + 立即购买(始终固定)
```

---

## 主图区(顶部约 1 屏)

| 元素 | 规则 |
|---|---|
| 比例 | 1:1 主图 |
| 轮播 | 自动轮播(3-5 秒)+ 手动滑动 |
| 视频 | 第一张可以是商品视频(自动播放 muted)|
| 角标 | 左上"自营 / 京东保" / 右上"分享 / 收藏" |
| 360° | 部分商品支持 360° 旋转(双指拖)|

---

## 价格区(主转化触发)

```
¥299.00       原价 ¥399.00
              限时秒杀  剩余 02:34:12
🎁 领券立减 ¥30          [领取]
```

**核心要素**:
- 主价:`Price.Large` (28pt) bold + 京东红 + number 字族
- 划线价:`Price.Strikethrough` 灰
- 限时倒计时:[FlashSaleTimer]
- 优惠券:[CouponCard 浅色版]

## SKU 选择(规格)

直接展示在 PDP 上(可选项),也可以点开 Sheet 选择:

```
颜色  : [● 深蓝] [○ 浅灰] [○ 经典黑]
尺寸  : [○ S] [● M] [○ L] [○ XL]
```

未选规格时加购 → 弹 SKU 选择 Sheet。

## 底部固定栏(主 CTA)

```
┌─────────────────────────────────┐
│ [客服] [店铺] [收藏] | 加购物车 | 立即购买 │
└─────────────────────────────────┘
```

- 始终固定底部(考虑 safe area)
- 主 CTA "立即购买"右侧 = primary
- "加入购物车"左侧 = secondary
- "客服 / 店铺 / 收藏" = 图标按钮(IconButton)

> **关键设计**:加购在左,立购在右(右手用户的拇指更易触达右侧)。两个并排但**不**都是 primary(详见 [[../../../../foundations/components-base/action/button/donts.md#multiple-primary]])

## 关键 KPI

| KPI | 当前 | 目标 |
|---|---|---|
| 加购率 | ~ % | ↑ 1% |
| 立购率 | ~ % | ↑ 0.5% |
| 收藏率 | ~ % | ↑ |
| PDP 平均停留时长 | 30-60s | 维持 |

---

## 状态完整性

- ✅ 默认
- ✅ 加载中(Skeleton 还原模块结构)
- ✅ 已下架(灰色蒙层 + "商品已下架"提示)
- ✅ 库存为 0(加购按钮变 disabled + "缺货" / "加入心愿单")
- ✅ 网络异常(错误页 + 重试)

---

## 反例

| ❌ 反面 | 解释 |
|---|---|
| 加购按钮和立购按钮都是 primary | 用户认知负担 / 转化率 -3.2% |
| 主图区超过 1.5 屏 | 看不到价格 |
| SKU 选择在底部需要滚动 | 决策路径长 |
| 优惠券领取要跳转新页 | 中断 PDP 浏览 |
| 详情图过大(超 5 屏) | 用户流失 |
