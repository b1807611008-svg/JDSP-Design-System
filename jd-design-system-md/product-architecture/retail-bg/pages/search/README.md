# 搜索页 · Search

> 用户**主动找货**的入口。**搜索的核心矛盾是"快速找到 vs 引导探索"**——精准搜索要快,长尾搜索要给推荐。

---

## 搜索 3 个阶段

```
1. 搜索前(空搜索框)→ 历史 + 热搜 + 发现
2. 输入中(有输入)→ 联想词 + Suggestion
3. 搜索后(结果页)→ 商品列表 + 筛选
```

---

## 阶段 1:搜索前

```
┌─────────────────────────┐
│ 🔍 搜你想要的            │  ← SearchBar(顶部固定)
├─────────────────────────┤
│ 历史搜索  [✕清空]        │
│ [iphone] [口红] [鞋子]    │
├─────────────────────────┤
│ 大家在搜                 │
│ 1 🔥 热搜词              │
│ 2 🔥 热搜词              │
├─────────────────────────┤
│ 为你推荐(双列商品流)     │
└─────────────────────────┘
```

---

## 阶段 2:输入中

输入"iph" → 显示联想词:
```
iphone 16
iphone 15 pro max
iphone 充电器
iphone 手机壳
```

- 联想词 ≤ 8 条
- 每条文字 + 搜索量(可选)
- 点击直接搜索

---

## 阶段 3:搜索结果

```
┌─────────────────────────┐
│ ← 🔍 iphone 16  [搜索]   │
├─────────────────────────┤
│ [综合][销量][价格↑↓][筛选]│  ← 排序筛选条
├─────────────────────────┤
│ 商品双列流(同首页)       │
│                         │
└─────────────────────────┘
```

- 排序:综合 / 销量 / 价格升降 / 评价
- 筛选:点击展开 sheet(详见 [[../../components-business/]])
- 无结果时:空状态 + 推荐相似 + "没找到 X"

---

## 关键 KPI

| KPI | 当前 | 目标 |
|---|---|---|
| 搜索成功率(有结果)| ~ % | 维持 |
| 搜索 → 点击 PDP 率 | - | ↑ |
| 搜索词长尾分布 | - | - |

---

## 状态完整性

- ✅ 搜索前(显示推荐)
- ✅ 输入中(联想词)
- ✅ 有结果(默认)
- ✅ 无结果(空状态 + 替代搜索词)
- ✅ 网络错误

---

## 反例

| ❌ 反面 | 解释 |
|---|---|
| 历史搜索不可清空 | 隐私担心 |
| 联想词超 8 条(挤压键盘)| 体验差 |
| 无结果直接显示空白页 | 用户流失 |
| 筛选条件多到记不住 | 决策瘫痪 |
