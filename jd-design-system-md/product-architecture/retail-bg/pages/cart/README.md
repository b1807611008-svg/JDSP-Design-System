# 购物车 · Cart

> 用户决策"买什么"和"买多少"的最后关口。**购物车的核心矛盾是"信息全 vs 主流程通畅"**——既要展示促销凑单,又不能挡住"去结算"。

---

## 模块清单

```
购物车
├── [Module] 顶部导航(标题"购物车"+ 编辑按钮)
├── [Module] 收货地址提示(可选)
├── [Module] 商品列表
│   ├── 店铺分组
│   ├── 商品项(CartItem)
│   └── 失效商品分组(末尾)
├── [Module] 凑单提示("再买 ¥30 享 8 折")
├── [Module] 推荐位(为你推荐)
└── [Bottom Bar] 全选 + 总价 + "去结算"
```

---

## CartItem 详解

```
┌─────────────────────────────────┐
│ ☑️  [商品图] 商品标题(2 行)      │
│        规格:深蓝 / M           │
│        ¥299.00         [- 1 +] │
└─────────────────────────────────┘
```

- 左侧:多选 checkbox
- 中间:商品图 + 标题 + 规格
- 右侧:价格 + 数量调整器
- 左滑:删除 / 收藏切换

详见 [[../../components-business/cart-item/]]

---

## 失效商品

商品下架 / 限购满 / 库存为 0:
- 整体灰化(opacity 50% via token,不是 hack)
- "失效"红色标识
- "找相似 / 删除" 操作
- 末尾分组,与正常商品分隔

---

## 底部固定栏

```
┌─────────────────────────────────┐
│ ☑ 全选  合计:¥1,234.00      [去结算] │
└─────────────────────────────────┘
```

- 主 CTA"去结算"= primary block(撑满右侧)
- 已选数量提示("已选 3 件")
- 总价 = 已选商品总和 - 优惠

---

## 编辑模式

点击右上"编辑":
- 全选变"批量删除 / 批量移收藏"
- 商品项不显示数量调整器,显示 checkbox
- 顶部 X 关闭编辑模式

---

## 关键 KPI

| KPI | 当前 | 目标 |
|---|---|---|
| 进入购物车率 | - | - |
| 结算转化率 | ~ 70% | ↑ 1% |
| 凑单完成率 | - | - |

---

## 状态完整性

- ✅ 默认(有商品)
- ✅ 空购物车("购物车空空如也,去逛逛"+ Joy 插画 + "去逛逛"按钮)
- ✅ 加载中
- ✅ 网络错误

**空购物车设计是关键**——不要直接显示空白页,要引导用户回流。

---

## 反例

| ❌ 反面 | 解释 |
|---|---|
| "去结算"按钮被推荐位挤掉 | 主转化阻塞 |
| 失效商品和正常商品混排 | 视觉混乱 |
| 数量调整器响应慢(> 200ms) | 体验差 |
| 删除无撤销 | 误删无法恢复 |
| 凑单提示阻塞结算 | 主流程被打断 |
