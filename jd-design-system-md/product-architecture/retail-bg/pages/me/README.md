# 我的(个人中心) · Me

> 用户身份、订单、会员、设置的聚合页。**"我的"页设计的核心矛盾是"功能多 vs 主信息突出"**——头像/订单/会员是主,其他都是次。

---

## 模块清单(从上到下)

```
我的
├── [Module] 顶部沉浸式背景(可个性化)
├── [Module] 用户信息(头像 + 昵称 + 等级)
├── [Module] 京豆 / PLUS / 优惠券 / 礼品卡 4 卡入口
├── [Module] 订单状态导航(待支付 / 待发货 / 待收货 / 待评价 / 售后)
├── [Module] 我的资产(收藏 / 关注 / 浏览历史)
├── [Module] 我的服务(客服 / 设置 / 帮助 / 反馈)
├── [Module] 推荐位(为你推荐)
└── [Bottom] Tab Bar(我的选中)
```

---

## 用户信息(顶部)

```
┌─────────────────────────┐
│  [头像]  昵称 ✨ PLUS   │
│         ID: 1234 等级 5 │
└─────────────────────────┘
```

- 头像:`avatar.l` (40pt) + 编辑入口
- 昵称:`heading.h3`
- PLUS 标识:`tag.brand-gold`

---

## 订单状态导航

```
┌─────────────────────────┐
│ 我的订单               全部> │
├─────────────────────────┤
│ [图]  [图]  [图]  [图]  [图]│
│ 待支付 待发货 待收货 待评价 售后│
│  3                          │
└─────────────────────────┘
```

- 5 个状态图标 + 文字
- 数字角标(`badge.number`):待办数量
- 点击跳转订单页对应 Tab

---

## 关键 KPI

| KPI | 关注 |
|---|---|
| PLUS 转化 | 是否点击会员区 |
| 优惠券领取率 | 是否点击优惠券入口 |
| 订单管理活跃 | 是否点击待支付 / 售后 |

---

## 状态完整性

- ✅ 已登录(默认)
- ✅ 未登录(显示"登录"+ 占位昵称 + 登录引导)
- ✅ PLUS 会员(黑金主题 + 专属标识)

---

## 反例

| ❌ 反面 | 解释 |
|---|---|
| 头像区被广告挤掉 | 主信息丢失 |
| 订单状态导航无角标 | 用户漏看 |
| 推荐位优先级超过订单 | 主功能被挤 |
| 设置项藏太深 | 用户找不到客服 |
